# Job Collector App

`Job Collector App` - это учебное веб-приложение для сбора, хранения и поиска вакансий.

Проект позволяет:
- загружать вакансии вручную через интерфейс;
- парсить вакансии из `fake-jobs`;
- загружать вакансии из `HeadHunter`;
- сохранять вакансии в SQLite-базу;
- искать, редактировать и удалять вакансии через веб-интерфейс и API;
- защищаться от повторного сохранения одних и тех же вакансий из HH по `url`.

## Назначение

Приложение нужно для локального сбора вакансий в одну базу данных с последующим поиском по ним через интерфейс.

Основной сценарий работы:
- пользователь запускает парсер HH;
- вакансии сохраняются в БД;
- при повторном запуске уже существующие вакансии не дублируются;
- затем пользователь ищет вакансии через форму поиска на главной странице.

## Структура проекта

- `app/main.py` - создание FastAPI-приложения и подключение роутов
- `app/routers/vacancies.py` - API для вакансий и парсинга
- `app/services/vacancy_service.py` - бизнес-логика работы с вакансиями
- `app/parsers/hh_parser.py` - асинхронный парсер HeadHunter API
- `app/parsers/asynco_parser.py` - парсер тестовых вакансий
- `app/models/vacancy.py` - SQLAlchemy-модель вакансии
- `app/schemas.py` - Pydantic-схемы
- `app/database.py` - подключение и инициализация БД
- `index.html`, `add.html`, `script.js`, `style.css` - интерфейс приложения

## Запуск

### 1. Установить зависимости

```bash
pip install -r requirements.txt
```

### 2. Настроить `.env`

В корне проекта должен быть файл `.env`:

```env
DATABASE_URL=sqlite:///./job_collector.db
HH_API_URL=https://api.hh.ru/vacancies
HH_API_EMAIL=your_email@example.com
FAKE_JOBS_URL=https://realpython.github.io/fake-jobs/
```

Поле `HH_API_EMAIL` используется в `HH-User-Agent`.

### 3. Запустить сервер

```bash
uvicorn app.main:app --reload
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000
```

## Интерфейс

Доступные страницы:
- `/` - главная страница со списком вакансий, поиском, кнопками запуска парсеров и редактированием
- `/add` - страница ручного добавления вакансии

На главной странице можно:
- запустить загрузку `fake-jobs`;
- запустить загрузку вакансий из `HH`;
- искать вакансии по ключевым словам;
- сортировать выдачу;
- удалять и редактировать записи.

## API

### Получить список вакансий

`GET /vacancies/`

Параметры:
- `q` - общий поисковый запрос
- `title` - фильтр по названию
- `salary` - фильтр по зарплате
- `company` - фильтр по компании
- `location` - фильтр по локации
- `sort_by` - сортировка, допустимые значения: `salary`, `created_at`
- `limit` - количество записей
- `offset` - смещение

Пример:

```http
GET /vacancies/?q=python+remote&sort_by=created_at&limit=10&offset=0
```

### Получить вакансию по ID

`GET /vacancies/{id}`

### Создать вакансию вручную

`POST /vacancies/`

Пример тела:

```json
{
  "title": "Python Developer",
  "company": "Example Company",
  "salary": 150000,
  "location": "Moscow",
  "published_date": "2026-04-10",
  "description": "Backend development",
  "url": "https://example.com/job/1",
  "source": "manual"
}
```

### Полностью обновить вакансию

`PUT /vacancies/{id}`

### Частично обновить вакансию

`PATCH /vacancies/{id}`

### Удалить вакансию

`DELETE /vacancies/{id}`

### Запустить парсер fake-jobs

`POST /vacancies/parse-fake`

Назначение:
- загружает тестовые вакансии;
- сохраняет новые записи в БД;
- пропускает уже существующие по `url`.

### Запустить парсер HeadHunter

`POST /vacancies/parse-hh`

Назначение:
- получает вакансии из `HH API`;
- загружает все доступные страницы ответа;
- сохраняет новые вакансии в БД;
- обновляет уже существующие вакансии по `url`;
- удаляет устаревшие HH-вакансии старше периода парсинга.

Пример ответа:

```json
{
  "saved": [],
  "received_count": 100,
  "created_count": 20,
  "updated_count": 80,
  "skipped_count": 0,
  "deleted_count": 5
}
```

## Как работает защита от дублей

Защита от повторной загрузки HH-вакансий реализована в двух слоях:
- на уровне приложения вакансия ищется по `url` перед сохранением;
- на уровне базы данных создан уникальный индекс на `url`, если `url` не `NULL`.

Это значит, что повторный запуск парсера не должен создавать дубли одних и тех же HH-вакансий.

## База данных

Используется таблица `vacancies` со следующими основными полями:
- `id`
- `title`
- `company`
- `salary`
- `location`
- `published_date`
- `description`
- `url`
- `source`
- `created_at`

## Примечания

- Поиск в интерфейсе работает через общий запрос `q` и сортировку.
- Вакансии из HH загружаются через публичный API `HeadHunter`.
- База данных создается автоматически при старте приложения.
